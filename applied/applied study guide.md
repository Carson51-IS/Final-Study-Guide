# Final Exam (Applied) Study Guide

You should be able to do tasks from the labs (or similar tasks—such as decryption with a different algorithm or encryption with an algorithm you have used) **individually**.

> This list is **not exhaustive**. It highlights practical skills to review.

---

## What to review

### Perform basic cyber defense

Be ready to apply foundational defensive practices covered in course labs and materials.

### Assess the threat of identification through aggregation

Understand concepts such as **k-anonymity** and how aggregated or released data can still identify individuals.

### Encrypt and decrypt text and files

Use algorithms that have been **described in class**, that **you have used**, or for which **sample code was provided**—including variants (e.g., different cipher or parameters) that mirror what you practiced.

### Hash files

Use **familiar** hashing algorithms or ones where **sample code was provided**.

### Apply and verify digital signatures

Use algorithms the course **discussed**, that **you have already implemented**, or for which **code was provided**.

### Digital certificates

Understand **what they are**, **what they assert**, and **how they fit** into trust models (e.g., PKI, CAs) at the level covered in class.

### Authentication and authorization techniques

Understand how systems **prove identity**, **grant access**, and related mechanisms you studied (e.g., factors, tokens, protocols at a high level).

### Adapt security-related code

Be able to **modify or extend** supplied examples—e.g., if you are given **steganography** code and asked to adapt it for a similar task.

### Social engineering

This is the **only possible comprehensive item** on the applied portion of the exam; be prepared for scenarios that tie multiple ideas together in a realistic way.

---

## Quick checklist

- [ ] Basic cyber defense tasks from labs
- [ ] k-anonymity / aggregation and identification risk
- [ ] Encrypt/decrypt (text or files) with class or lab algorithms
- [ ] Hash files with known or provided algorithms
- [ ] Sign and verify with class/lab/signature code you have seen
- [ ] Digital certificates (purpose, trust chain, key ideas)
- [ ] Authentication vs. authorization and common techniques
- [ ] Small code adaptations (e.g., steganography-style changes)
- [ ] Social engineering (comprehensive-style item)
